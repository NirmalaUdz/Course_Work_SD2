Splash Page

Technical Description

The splash page is designed using a combination of HTML, CSS, and JavaScript to create an engaging and informative introduction to the website.

HTML Techniques:

Tags: Used to create containers for different sections of the splash screen, such as the splash content and loader.

  • h2 Tags: for Usedheadings to provide a clear hierarchy of information.
  • li Tags: Used to list team members in a structured manner.
  • button Tag: Used to create an interactive button for the page editor.

    • CSS Techniques:

  • CSS Variables: Defined using :root for consistent theming across the page (--primary-color, --secondary-color).
  • Global Styles: Resets margins and paddings, sets box-sizing, and applies a default font and background color.
  • Ripple Effect: Adds a ripple animation for visual effect.
  • Loader Animation: Creates a water drop animation with a ripple effect.
  • Responsive Design Adjusts styles for smaller screens to ensure the layout remains user-friendly.

    • JavaScript Techniques:

  • Initial Setup: Sets the Splash Screen to display and hides the main content.
  • Mousemove event: Creats ripple effect when mouse pointer is moving
  • Timeout Function Fades out the splash screen after 4 seconds

    • Accessibility

    Using of Alt Text for Images. Color contrast between the background and other details to ensure the proper readability. Animations and effects using java script

    Content Page

    Technical Description

    HTML Techniques:

    The HTML structure uses semantic tags like header, nav, main, section, and footer to organize content logically and improve accessibility. Key elements include headings (h1, h2,h3), paragraphs (p), images (img with alt attributes), and lists (ul, li). These tags help create a well-structured, readable, and navigable document..

    CSS Techniques:

    The CSS styles enhance the visual appeal and responsiveness of the page. Flexbox and Grid Layout are used for flexible and complex layouts, respectively. Typography is styled with font and color properties, while media queries ensure the design adapts to different screen sizes. Custom classes and properties like margin, padding, and background-color are applied to maintain consistency and improve readability.

    Accessibility

    The page uses semantic HTML tags like header, nav, main, section, and footer to improve structure and navigation for assistive technologies. All images include alt attributes to provide descriptive text for screen readers. The aria-label attribute is used for interactive elements like the "Go to Top" button to enhance accessibility. Keyboard navigation is ensured for all interactive elements, and sufficient color contrast is maintained to improve readability for users with visual impairments.

    Challenges and Lessons Learned

    During the implementation, I encountered issues with how different browsers rendered certain CSS properties. To resolve this, I used vendor prefixes and tested the page across multiple browsers to ensure consistent appearance and functionality. It's crucial to test web pages on various browsers and devices early in the development process to identify and address compatibility issues.

    Creating a layout that adapts well to different screen sizes was challenging. I utilized Flexbox and Grid Layout along with media queries to create a responsive design that works on both desktop and mobile devices. Responsive design requires careful planning and testing. Using modern CSS techniques like Flexbox and Grid can simplify the process and improve the overall user experience.

    Ensuring the page was accessible to all users, including those with disabilities, required adding alt attributes to images. Accessibility should be a priority from the start. Regularly testing with accessibility tools can help identify and fix issues early, making the web page more inclusive.
